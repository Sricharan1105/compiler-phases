1. Lexical Analysis

Converts source characters into a stream of tokens: keywords, identifiers, literals, operators, punctuation. Removes whitespace and comments.

2. Syntax Analysis

Parses the token stream against the C grammar. Builds an Abstract Syntax Tree (AST) showing the hierarchical program structure.

3. Semantic Analysis

Checks meaning: type correctness, scope rules, undeclared identifiers. Populates the Symbol Table with each variable's type and scope.

4. Intermediate Code

Converts the AST into Three-Address Code (TAC) — a machine-independent IR that bridges high-level code and machine instructions.

5. Code Optimization

Improves the IR: constant folding, dead code removal, loop optimisation. Same behaviour, fewer / faster instructions.

6. Code Generation

Maps optimised IR to target assembly instructions. Handles register allocation and instruction selection for the final output.
